# study-quest プロジェクトルール

## 下書き（ブランチ）運用 — 2026-07-16 導入

並行作業の渋滞・上書き事故をなくすため、main 直接編集をやめて下書き方式にする。

### 基本の流れ

1. 作業を始めるとき、main を直接編集しない。作業テーマごとに下書きブランチを作る
   - ブランチ名: `draft/<テーマ>`（例: `draft/grade1-pack`）
2. 並行セッションで走る場合は worktree（分身フォルダ）で作業場所ごと分ける

   ```
   git worktree add ../study-quest--<テーマ> -b draft/<テーマ>
   ```

   分身フォルダは `C:\Users\yousu\projects\study-quest--<テーマ>` に置く
3. 合流（マージ）の前に必ず検品する（該当学年で1バトル通し＋クラウド保存の動作確認）
4. 検品合格 → ユーザーに一言確認してから main へマージ → 使い終わった worktree とブランチは削除

   ```
   git worktree remove ../study-quest--<テーマ>
   git branch -d draft/<テーマ>
   ```

### 公開のルール

- 本番（GitHub Pages）が更新されるのは main を push した時だけ
- push は毎回ユーザーへ明示確認（グローバルルールどおり。「続き」を push 許可に拡大解釈しない）

### 衝突を避けるための交通整理

- **index.html を書き換える作業は同時に1本まで**（巨大単一ファイルのため必ず衝突する）
  - これは暫定ルール。`spec/構造分離-設計メモ.md` のデータ分離が完了したら緩和する
- spec/ への設計メモ追加など、新規ファイルだけの作業は衝突しないので並行自由
- 作業開始時に `git worktree list` と `git branch` で、ほかに何が走っているか確認してから着手する

### 例外（経過措置）

2026-07-16 時点ですでに main 直で走行中だった作業（学年拡張・難易度底上げ）は、
そのまま main で完走してよい。次の新規作業からこのルールを適用する。
