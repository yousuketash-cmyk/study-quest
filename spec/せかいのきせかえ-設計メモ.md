## 決定事項（2026-07-20 本人確定）

1. ことばの着せ替え＝**要所だけ辞書化**（ホームのメニュー文言＋バトルの要所10〜20語＋勝利メッセージ）
2. キャラの絵＝**全部共通**（171枚をそのまま使う。枠・色・肩書きだけテーマ連動。新規発注なし）
3. 第1弾＝**モック通り3テーマ**（ぼうけん整え版・コスモ・ゆめかわ）＋「じゅんび中」枠（きょうりゅう・スイーツ）
4. 原則＝**機能は全世界共通・呼び名だけ変える**（世界によってボタンが消えることはない）

## ねらい

- 男の子寄りの世界観を「自分の世界を選ぶ」方式に広げる（ランドセルの色選び）
- カスタマイズ構想の「子ども向けの柱」。親向けの柱（とっくんセット等）とは別テーマで進める

## 実装の土台（調査済み・2026-07-20）

有利な材料が2つ既にある：

1. **色は :root の CSS変数に一元化済み**（index.html 冒頭 `--bg1 --bg2 --card --ink --accent` 等）
   → テーマ切替＝変数の差し替えが基本。加えて body クラス（`theme-cosmo` 等)で
   部品単位の上書き（カードの白/黒など）を足す
2. **表示文言の置換機構が既にある**（kanjify / kanjifyDom：学年に応じて
   ひらがな→漢字へ置換。index.html:1543 付近）
   → ことば辞書は同じ方式で themify を作れば良い。**適用順はテーマ置換→漢字置換**
   （テーマ語もひらがな表記で辞書に載せ、低学年の漢字変換と両立させる）

保存まわり：プレイヤーごとのセーブ D は localStorage＋saveData()→schedulePush() で
Supabase へ同期済み。**D.theme を1項目足すだけで端末またぎ・クラウド保存が自動で付いてくる**
（D.books 方式の前例どおり）。

## データ設計

```js
// テーマ台帳（エンジン内に定数で持つ。問題データとは無関係なので data/ には置かない）
const THEMES = {
  bouken: { name:"ぼうけん", sub:"ゆうしゃ・モンスター", icon:"⚔️", ready:true,
            vars:{...}, words:{...}, ranks:[...] },   // 現行＝既定値。整え版の配色に更新
  cosmo:  { name:"コスモ",   sub:"うちゅう・ネオン",     icon:"🌌", ready:true, ... },
  yume:   { name:"ゆめかわ", sub:"まほう・パステル",     icon:"🌸", ready:true, ... },
  dino:   { name:"きょうりゅう", icon:"🦕", ready:false },   // じゅんび中＝選択不可で表示
  sweets: { name:"スイーツ",     icon:"🍰", ready:false },
};
// プレイヤーごとの保存： D.theme = "bouken" | "cosmo" | "yume"（無ければ bouken）
```

- 既存プレイヤーは D.theme 未設定＝ぼうけん のまま。**移行処理は不要**（何も起きないのが正）
- 兄弟それぞれ別テーマ可（D はプレイヤー単位のため自動的にそうなる）

## ことば辞書（要所だけ・各テーマ15語目安）

対象にする語（ぼうけん→他テーマの例）：

| 共通キー | ぼうけん | コスモ | ゆめかわ |
|---|---|---|---|
| 出発ボタン | ぼうけんに でる | たんさに でる | おでかけ する |
| 敵 | モンスター | エイリアン | モンスター（変えない） |
| たおす | たおす | たおす | なかよく なる |
| ミッション | きょうの ミッション | きょうの ミッション | きょうの おてつだい |
| 図鑑 | なかま ずかん | なかま ずかん | なかま ずかん |
| 肩書き（例） | みならい ゆうしゃ | たんさくしゃランク | みならい まほうつかい |
| 勝利 | やったね！ かった！ | ミッション クリア！ | なかよく なれたよ！ |

- 「たおす→なかよくなる」系は**表示文言のみ**。内部の勝敗判定・記録の言葉（親画面・stats）は変えない
- 親画面（おうちの人）とadmin・stats は**テーマ適用外**（読む人が親のため。常にぼうけん語彙＝現行のまま）
- 問題文・選択肢は絶対に触らない（kanjify と同じ原則）

## 画面

1. **せかいえらび**：新規プレイヤー作成フロー（学年・教科書えらびの並び）に1画面追加＋
   ホームの設定から変更可。「あとから いつでも かえられるよ」を明記。じゅんび中2枠も表示
2. **ホーム**：モック④「ぼうけん整え版」の整理をベースに、テーマで色・文言・肩書きが変わる
3. **バトル**：色（背景・ボタン・ダメージ表示色）＋要所の文言のみテーマ連動。
   演出（エフェクト）の差し替えは第2弾（モックにあるが第1弾では色連動まで）

## やらないこと（第1弾）

- テーマ専用キャラ・専用ガチャ・専用図鑑（絵は全部共通）
- バトル全文言の辞書化／効果音・エフェクトの差し替え
- きょうりゅう・スイーツの中身（枠だけ見せる）
- 親画面・admin・stats へのテーマ適用

## 実装時の決定（2026-07-20 実装・机上検品済み）

- **ぼうけんは現行の見た目を完全維持**（モック④の「整え」は見送り。検品条件
  「既存セーブで見た目不変」を優先。:root の既定値＝ぼうけんそのもの）
- コスモの語彙は控えめ（たんさに でる／たんさくしゃランク／🚀）。モックが
  コスモでも「たおす」を残していたのに合わせ、敵まわりの言葉は共通のまま
- ことばの置き換えは kanjifyDom と同じ TreeWalker 方式（themifyDom）。
  もとの文は WeakMap に控えを取り、テーマを戻したら原文復元。
  除外＝ .qcard（問題文・選択肢・数字パッド）と #reportBody（おうちの人へ）
- 勝敗オーバーレイ（winScreen/dayDone/loseScreen）は表示直前に themifyDom を通す
- 検品済み：新規作成でゆめかわ選択→1年生1バトル勝利／テーマ3種切替と原状復帰／
  リロード後の維持／theme なし旧セーブ→ぼうけん既定／親画面に置換語の混入なし／
  コンソールエラーゼロ
- **クラウド検品も合格（2026-07-20）**：検品用あいことば kenpin-theme-0720 で
  作成→テーマがクラウド保存(theme:"yume")→端末またぎ復元（localStorage全消去→
  あいことばで入り直し→色・ことばとも復元）→テーマ変更の再同期(cosmo)まで実機確認。
  検品データは sq_delete_player＋Studio SQL で全削除（remaining_families=0 確認済み）
- **残り＝スマホ実機のコントラスト確認（特にコスモ）のみ。公開後の確認でよい**
