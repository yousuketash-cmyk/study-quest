# 構造分離（データの別ファイル化）設計メモ（2026-07-16 合意）

並行作業の衝突を根絶するための構造改修。
「1つの建物に学年別フロア」（学年拡張-設計メモ.md）の方針はそのまま守り、
**引き出しの中身（問題データ）だけを別ファイルの倉庫に出す**。

━━━━━━━━━━━━━━━━━━━━━━
## 1. 目的（なぜやるか）

- 現状はゲーム全体が index.html 1ファイル（約31万字）に同居しており、
  「問題を作る作業」と「仕組みを直す作業」が必ず同じファイルを取り合う
- データを別ファイルに出せば、問題作成系の作業はデータファイルだけ触る
  → index.html を触る作業と衝突しなくなり、下書きの合流（マージ）も一瞬で終わる

## 2. 分離の方針

- index.html ＝ エンジン（ゲームの仕組み・画面・保存）だけ残す
- data/ フォルダ ＝ 問題データの倉庫。読み込みは `<script src="data/○○.js">`
  （GitHub Pages でそのまま動く。ビルド工程は増やさない）

### 倉庫の区切り案（作業がぶつからない単位で切る）

| ファイル | 中身 |
|---|---|
| data/pack-g1.js | 1年生の引き出し（算数単元・漢字・例文・単元カレンダー） |
| data/pack-g3.js | 3年生の引き出し（同上） |
| data/pack-g6.js | 6年生の引き出し（同上） |
| data/subjects-extra.js | りか・しゃかい・えいご等の在庫（難易度底上げで5倍化予定の部分） |
| data/advanced.js | ⚡おうよう問題（2階部分） |

区切りの粒度は着工時に実物を見て微調整してよい。
原則は「同時に走りそうな作業が別ファイルになるように切る」。

## 3. 手順（着工条件つき）

1. **着工条件：走行中の並行作業（川口対応・難易度底上げ）が main に着地してから**
   - 分離前に index.html の大移動をすると、走行中作業との合流が大事故になるため
2. `draft/structure-split` ブランチ＋worktree で作業（CLAUDE.md の下書き運用どおり）
3. 移動は**機械的な引っ越しのみ**。ロジック変更・問題の追加修正を混ぜない
   - 「移動だけのコミット」と「変更のコミット」を絶対に分ける（検品を楽にするため）
4. 検品：全学年（1・3・6年）で1バトル通し＋クラウド保存＋見守り画面の表示確認
5. 合格 → ユーザー確認 → main へ合流 → push（要明示確認）

## 4. 分離後にできるようになること

- 並行3セッションが現実的になる（例：1年生の問題作成／6年生の問題作成／エンジン改修）
- 難易度底上げ（在庫5倍化）の大量データ作成を、エンジン作業と完全並行できる
- 「一括作成→検品→公開」の検品対象がデータファイル単位になり、差分が見やすくなる

## 5. やらないこと

- 学年ごとの別アプリ化（コピー方式は引き続き禁止）
- ビルドツール・フレームワークの導入（静的ファイルのまま）
- この分離作業のついでの機能追加（引っ越しと模様替えを混ぜない）
