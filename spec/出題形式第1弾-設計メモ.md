# 出題形式の幅だし 第1弾（○×クイズ＋あなうめ）設計メモ

作成：2026-07-20 ／ 背景：形式が実質3系統（4択・数字入力・手書き）で4択に偏っており、
体感の「同じ景色」を崩したい。本人決定＝第1弾は ○×＋あなうめ（2026-07-20）。

## 1. ○×クイズ（自動生成・全教科のスパイス）

### ねらい
- **データ追加ゼロ**で全教科に新形式を展開する。既存の問題から「答えの提示→合ってる？」を作る
- テンポの良い2択で、低学年でも指が迷わない

### 作り方（oxify：既存問題の変換）
- バトルの出題直前（loadQ）で、変換できる問題なら **約22%の確率で ○×に変換**
- 変換ルール：
  - 4択系（choices あり）：正解 or ハズレ選択肢を「こたえ」として提示
    →「こたえは『6本』で あってる？」＋ ○ × の大ボタン2つ
  - 計算系（数字入力）：「7 + 3 ＝ 10 → あってる？」。ハズレは ±1〜3 か 位取りゆらし
  - 正誤は五分五分。ハズレ提示のときは その問題のハズレ選択肢から選ぶ（もっともらしさ担保）
- **変換しないもの**：かきとり（手書き）・よみとり長文・もじ（絵あわせ）・復習の出しなおし問題
- ○×は **復習プールに入れない**（4択と同じ扱い）・**ちからだめしには出さない**（バトル専用のスパイス）
- 記録（log の cat・ミッション・親レポート）は 元の問題の教科/分野を引き継ぐ

### 画面
- 既存 .choices の器に「○」「×」の特大2ボタン（緑/赤・choice クラス流用）
- 1年生の よみあげ（speech）対応：「こたえは 『○○』で あってるかな？」

## 2. あなうめ（ことばのきまり）＝こくごの新しいとき方

### ねらい
- 文の中の空欄を埋める形式。助詞・かなづかい・つなぎことば・敬語という
  **今のスタディクエストに無い学習領域**を同時に獲得する

### 組み込み方（よみとり＝DOKKAI と同じ型）
- データ：`data/kotoba-g1.js` 〜 `kotoba-g6.js`（新設・KOTOBA_G1〜G6）
- 引き出し：GRADE_PACKS に kotoba を追加 → applyGradePack で KOTOBA に入る
- こくごの とき方えらびに 「✏️ ことばのきまり」を追加（よみとりの並び）
- 漢字あそび中にも 低確率（15%）で混ぜる（よみとりの 20-30% 混入と同じ発想・控えめに）

### データ形式（クイズ形式 {t,a,w} を踏襲・エンジン改修最小）
```js
const KOTOBA_G1=[
  {t:"わたし［　］ がっこうへ いく。", a:"は", w:["を","へ"]},
  {t:"えほん［　］ よむ。",           a:"を", w:["は","が"]},
  ...
];
```
- 表示は既存4択の器のまま：本文＋hint「［　］に はいるのは どれ？」
- ［　］は目立つ下線スタイルにする（qtext 内の置換・小工事）

### 学年別の中身（各15〜20問・文はオリジナル・教科書の文章は使わない）
| 学年 | 領域 |
|---|---|
| 1年 | は/を/へ・かなづかい（じ/ぢ・長音）・小さい「っ」 |
| 2年 | かたかなで書くことば・丸/点/かぎ・こそあどことば |
| 3年 | つなぎことば（だから/しかし等）・修飾のことば |
| 4年 | つなぎことばの使い分け・慣用句/ことわざ入門 |
| 5年 | 敬語（尊敬/けんじょう/ていねい）・和語漢語外来語 |
| 6年 | 敬語の使い分け・熟語の組み立て・呼応の副詞 |

- 表記ルールは既存準拠：低学年はひらがな分かち書き・kanjify で自動漢字化に任せる
- 検品：answer が wrongs に混じっていない・重複なし を機械チェック＋実機バトル通し

## やらないこと（第1弾）

- ○×の ちからだめし出題・かきとりへの適用
- ことばのきまりの ちからだめし対応（在庫が育ってから）

## 検品（合流前）
- ○×：全教科で出ること・正誤五分五分・記録の教科が正しい・復習プールに入らない
- あなうめ：1年と5年で1バトル通し・［　］表示・kanjify との両立
- 既存形式に退行がないこと（4択・数字・かきとり・よみとり各1問）
- クラウド保存の動作確認（セーブ形式は不変のため軽め）
