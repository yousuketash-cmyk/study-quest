### カレンダー倉庫（data/book-cal-*.js・1ファイル=教科×出版社）
標準（★）の出版社はカレンダー不要（ベースの単元配列をそのまま使う）。それ以外を倉庫化する：

```js
var BOOK_CAL = (typeof BOOK_CAL === "undefined") ? {} : BOOK_CAL;
BOOK_CAL.math = BOOK_CAL.math || {};
BOOK_CAL.math.tosho = {
  1: [ {base:"g1u1", name:"なかまづくりとかず", term:1, s:[4,5]}, ... ],
  2: [ ... ], 4: [ ... ], 5: [ ... ], 6: [ ... ],
};
```

- 1行＝1単元：`{base:"<ベース単元id|null>", name:"<その社の単元名>", term:1|2|3, s:[月,日]}`
- `base` は spec の「流用する既存gen/qs」列そのまま。「対応genなし/qsなし」は `base:null`（エンジンが出題対象外にする）
- 同じ base を複数行で使ってよい（分割流用）。1行に2genの相乗りは**最初の1つだけ**を base にし、`/* 相乗り: u13 */` コメントを残す（エンジンv1は1単元1baseで割り切る）
- ファイル一覧（12本）：
  - 算数: book-cal-math-tosho.js（3年は作らない=川口の MATH_UNITS_G3_KAWA を流用）／-gakuto／-kyoshup／-dainippon／-nichibun（各1〜6年）
  - 理科: book-cal-sci-tosho.js（3年は SCI_UNITS_G3_KAWA 流用）／-gakuto／-kyoshup／-keirin／-shinkyo（各3〜6年）
  - 社会: book-cal-soc-tosho.js（3年は SOC_UNITS_G3_KAWA 流用）／-nichibun（各3〜6年）

### エンジン（index.html applyGradePack）
- 解決関数 `resolveBookUnits(subject, pub, grade, baseUnits)`：
  - BOOK_CAL に該当が無ければ baseUnits をそのまま返す（標準・未対応学年のフォールバック）
  - あれば各行を `Object.assign({}, base単元, {id:"bk_"+pub+"_g"+grade+"_"+(i+1), name, term, s, adv:(算数のみ base id)})` で複製。base:null は gen/qs なしの器（出題除外）
  - 東書の3年だけ特例：川口の *_KAWA 配列をそのまま返す（kwu9 の拡張gen等を活かす）
- id は `bk_<pub>_g<学年>_<連番>`（決定的・既存idと衝突しない）。記録（log の e.u）はこの id で貯まる
- ⚡おうよう：算数は adv=base id 経由で MATH_ADV_G* を引く（川口方式と同じ）。base:null の単元は応用も無し
